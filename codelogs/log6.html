Code Log 6: Reference Quest

  1. What did you look up?

    <details>

  2. What sites did you consult? Include links to the specific pages on those 4 resources.
    • CSS Tricks: https://css-tricks.com/using-styling-the-details-element/
    • web.dev: https://web.dev/learn/html/details#the_htmldetailselement_interface
    • MDN: https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/details
    • W3Schools: https://www.w3schools.com/tags/tag_details.asp
  3. What are the differences in the information presented on each site?
    CSS-Tricks

    Talks more about how to style the <details> element and gives practical examples of changing the look.

    web.dev

    Focuses on how the element behaves when the user opens or closes it and brings up accessibility points.

    MDN

    Talks about the technical info, examples, attributes, and browser support.

    W3Schools

    More basic and only shows the quick syntax without going very deep.

  4. What features do you think a code reference page should offer?

    I think a good code reference page should give clear examples and explain how the element behaves by default. It also helps when the page shows browser support, accessibility notes, and a couple of real use cases, not just the basic syntax. Having short explanations, sample code, and quick copy-paste sections makes it easier to understand and apply in real projects.

  5. Do any of your 4 resources stand out as particularly good or bad?

    It kind of depends on how deep someone wants to go into the topic. If the person wants something quick and simple, W3Schools can actually feel more useful because it gets straight to the point. MDN has great content, but personally I don’t love the overall layout because it makes the page feel a bit heavy and harder to read. Web.dev also has a lot of good information, but the texts are really long and can get overwhelming. CSS-Tricks is the one that catches my attention the most, because the layout is cleaner and easier to follow, and the examples feel more practical.
