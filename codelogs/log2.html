Code Log 2: CSS Measuring Units

In CSS, we use two main types of units to define sizes: absolute and relative. Absolute units don’t change based on screen size or zoom. They’re fixed values. Relative units adjust depending on the context, like the parent element or the size of the screen. If you want your layout to be more flexible and responsive, relative units are usually the better option.

  • %

    The percent sign is a relative unit. It calculates the value based on the size of the parent element. For example, if I set width: 50%, the browser gives the element half the width of its container. But this changes depending on the property. For width or height, the reference is the parent. For padding or margin, it usually uses the element’s own width. So you have to test it and see how the browser reacts in each case.

  • em and rem

    These are both relative to font size. em is based on the font size of the parent element. That means it can stack if you have nested elements, which sometimes makes things harder to control. rem is based on the root font size, usually the tag. It’s more stable and predictable, so using rem is appropriate for consistent spacing or font sizes across the page.

  • px

    Pixels are an absolute unit. They represent a fixed size on the screen. One pixel is one dot on the screen. This means that if you set something to 100px, it will always be 100 pixels wide, regardless of the screen size or resolution. However, with high-DPI screens, a pixel might not correspond to a physical dot, so it can look different on various devices.
